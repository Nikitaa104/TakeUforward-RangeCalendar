# 🗓️ Interactive Wall Calendar Component

An interactive, responsive wall calendar component built with **Next.js and React**, inspired by the aesthetic of a physical wall calendar.
This project focuses on combining **clean UI design with real user interaction**, including date range selection and contextual note-taking.

---

## ✨ Features

### 📅 Date Range Selection

* Select a start and end date directly from the calendar grid
* Clear visual distinction between:

  * Start date
  * End date
  * Dates in between
* Handles edge cases like reverse selection (end before start)

### ✍️ Integrated Notes System

* Add notes for:

  * Entire month
  * Selected date ranges
* Notes dynamically update based on selection
* Data persists using **localStorage**

### 🎨 Wall Calendar Aesthetic

* Inspired by a real-world wall calendar layout
* Includes:

  * Hero image as a visual anchor
  * Structured calendar grid
  * Dedicated notes section
* Clean and balanced UI with attention to visual hierarchy

### 📱 Fully Responsive Design

* Desktop: Multi-section layout (image + calendar + notes)
* Mobile: Stacked layout with touch-friendly interactions
* Optimized for different screen sizes

### ⚡ UX Enhancements

* Smooth date selection interactions
* Hover and active states for better feedback
* Clear and intuitive user flow

---

## 🧱 Tech Stack

* **Framework:** Next.js (React)
* **Styling:** Tailwind CSS (or your styling approach)
* **State Management:** React Hooks
* **Persistence:** localStorage (client-side only)

---

## 🧠 Key Implementation Details

* Modular component architecture for maintainability
* Custom logic for:

  * Calendar grid generation
  * Date comparison and range calculation
* Custom hook for handling persistent state
* Separation of UI and business logic for clarity

## 🎯 Design Approach

The goal of this project was not just to replicate a calendar UI, but to create a **usable and interactive component** that feels like a real product.

Key focus areas:

* Balancing aesthetics with functionality
* Making interactions intuitive (especially range selection)
* Ensuring responsiveness without breaking usability
* Keeping the codebase clean and modular
